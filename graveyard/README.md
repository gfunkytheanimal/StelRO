# graveyard/

Retired threads, experiments, and framings that didn't survive contact with
the data or the argument. Kept — not deleted — because the dead ends are
often the clearest explanation of why the current approach is what it is.

Each entry should record:
- **When** it was retired (ISO date)
- **What** it was trying to do (one sentence)
- **Why** it was retired (one sentence, honest)
- **Where** the code / notes live (path under `graveyard/` or commit SHA)
